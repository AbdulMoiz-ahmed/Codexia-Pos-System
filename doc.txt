Inventory & Finance Management System (IFMS)
1. Project Overview
The Inventory & Finance Management System (IFMS) is a role-based business management solution designed to handle sales, purchases, inventory tracking, and complete financial accounting. The system supports credit-based sales, customer debt tracking, vendor management, and automated notifications.
The system will be modular, secure, and scalable, suitable for small-to-medium businesses.

2. Core Objectives
Manage inventory (stock in / stock out)
Enable controlled sales and purchases based on user roles
Maintain accurate financial ledgers (credit, debit, assets, expenses)
Support credit sales and customer debt tracking
Generate real-time reports (inventory, profit & loss, ledgers)
Send SMS notifications to customers for outstanding dues

14. E-commerce Storefront Module (New)
This module adds a public-facing website where customers can browse the inventory and place orders.
14.1 Storefront Features (Customer Side)
Product Catalog: Responsive web view showing product name, image, description, and price.
Availability Status: Real-time "In Stock" or "Out of Stock" labels (synced with Module 4).
Shopping Cart: Add/Remove items and calculate totals.
Guest Checkout: Simple form for Customer Name, Phone Number, and Shipping Address.
Payment Method: Strictly Cash on Delivery (COD) (as requested).
Order Confirmation: Unique Order ID generated for the customer.
14.2 Admin Control & Web Management
Website Visibility Toggle: A specific "Show on Website" checkbox in the Item Creation/Editing screen. Only items checked by the Admin will appear on the storefront.
Web Order Management: A dedicated "Online Orders" queue where staff can approve, ship, or cancel orders.
14.3 Inventory & Finance Integration
Automatic Reservation: When a web order is placed, the system can "soft-reserve" stock to prevent overselling.
Transaction Flow:
On Order Placement: Order status = "Pending". No ledger entry yet.
On Delivery/Completion: * Inventory ‚Üí Debit (Stock Out)
Cash/Customer Ledger ‚Üí Credit (Sale)
(This maintains the Double Entry Accounting rules in Section 8).
